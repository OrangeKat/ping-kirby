Variables:

A variable is a way to store information so you can use it later. Think of it as giving a name to a particular piece of data. Variables help us remember and use data without having to retype it every time. It's like writing down your friend's phone number so you don't have to remember it every time you want to call them.

Variables have a type and a value: like their name suggest, can change over time; however, they CANNOT change their type.

A type may be an object (more information on this later on) or a primitive. Here is a list of all primitives in Java:

  • byte (number, 1
  • short (number, 2
  • int (number, 4 bytes)
  • long (number, 8 bytes)
  • float (float number, 4 bytes)
  • double (float number, 8 bytes)
  • char (a character, 2 bytes)
  • boolean (true or false, 1 byte)

Java is a strong typed language, which means variables need to be defined before we use them

Let's take a look at the following variable declarations

int kir;
int by = 8;
kir = 3;
by += 2;

int kir;
int by = b * 3;

The variables "kir" and by are initiliazed with type int. by is also given an initial value, equal to 8

kir = 3;

The variable "kir" is given a value, equal to a + 3. Because the variable was already declared, no need to specify the type, just type its name.

by += 2

We are adding 2 to the variable "by" here: the += symbol means that we take the base value of the "by" variable and add the right side to it.

Functions:

Functions (or methods, both conventions are correct) can be seen like machine turning one or multiple (or maybe none at all) inputs into zero or one output.

In Java, all function definitions must be inside classes. Let's look at a function declaration:

public class Tuto {
    public static int sum(int kir, int by) {
            return kir + by;
    }
}

We will ignore for now the details of the declaration of the class Tuto Let's look at the function declaration:

  • The public keyword indicates that everyone can access this function
  • The static keyword indicates that the function can be called without an instance of the class Tuto: the function sum can be called using Tuto.sum(a, b);
  • The int keyword indicates the return type of the function (what the function outputs): here it is an int (integer number)
  • sum is the name of the function int a and int b are the arguments of the function (the function input): here it is two integers. A function can take any number of input, including 0: in that case, you write void in the function declaration
  • Finally, the opening brace { declares the body of the function, where the magic happens.

Now let's take a look at the function body:

return kir + by;

The return keyword indicates the end of the function, and its output. Here, we are returning the sum of the kir and the by variables passed as input.
